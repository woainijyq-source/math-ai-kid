# 首轮内部试玩执行顺序

目标：用一轮 15 到 20 分钟的内部试玩，验证孩子是否愿意开始、愿意继续、以及最吃哪一种反馈。

## 开始前

1. 启动项目：`npm run dev`
2. 单独确认基础链路：`npm run smoke`
3. 打开试玩记录模板：[playtest-notes.md](C:/Users/Administrator/.openclaw/workspace/math-ai-kid/docs/playtest-notes.md)
4. 打开试玩检查单：[playtest-checklist.md](C:/Users/Administrator/.openclaw/workspace/math-ai-kid/docs/playtest-checklist.md)

## 试玩顺序

### 第 1 段：AI 对手型

目标：验证孩子是否能立刻进入状态，并愿不愿意继续下一轮。

1. 从首页进入 `AI 对手型`
2. 不解释太多，只说一句：
   “你和 AI 轮流拿月石，看看能不能找到更稳的办法。”
3. 让孩子自己选第一步
4. 完成一轮并进入结果页
5. 重点记录：
   - 是否能快速理解规则
   - 是否会停下来观察
   - 完成后是否主动问“再来一次行不行”

### 第 2 段：AI 共创型

目标：验证孩子是否愿意主动提出规则，而不是只接受题目。

1. 从首页进入 `AI 共创型`
2. 引导语只说一句：
   “你来定一个新规则，看看 AI 能不能接住。”
3. 优先让孩子先自由说
4. 如果卡住，再提示使用“规则起手式”或规则碎片
5. 完成一轮并进入结果页
6. 重点记录：
   - 是不是愿意自己开口定规则
   - 更自然的是自由说，还是点碎片
   - AI 反向出题时，孩子会不会觉得有趣

### 第 3 段：AI 剧情型

目标：验证孩子是否能感受到“我的选择改变了故事”。

1. 从首页进入 `AI 剧情型`
2. 引导语只说一句：
   “你来决定接下来发生什么。”
3. 连续走完一轮分支
4. 重点记录：
   - 会不会认真比较选项
   - 是否会提到“这个会发生什么”
   - 结果页能不能让孩子感到自己的选择留下了变化

## 结束后必须看

1. 回到首页，确认世界状态有变化
2. 打开奖励页，确认结算能读懂
3. 打开家长页，确认摘要和刚才的试玩一致
4. 立刻填写 [playtest-notes.md](C:/Users/Administrator/.openclaw/workspace/math-ai-kid/docs/playtest-notes.md)

## 记录优先级

优先记这些，不要平均用力：

1. 哪个原型最容易开始
2. 哪个原型最容易让孩子继续下一轮
3. 哪种反馈最能让孩子停下来继续看
4. 第一次明显走神是在哪一刻

## 当场不要做的事

- 不要解释太多规则
- 不要提前告诉孩子“正确策略”
- 不要把试玩做成问答测试
- 不要在试玩过程中频繁打断追问

## 本轮结束判断

如果出现下面任意一条，下一轮优先改体验，不急着扩大试玩：

- 孩子开始前就不知道该点哪里
- 完成一轮后没有明显“变化被看见”的感觉
- 家长页和刚刚的试玩不一致
- 某个原型明显比另外两个弱很多，孩子不愿进入
