# AI 训练系统作战手册

## 1. 文档目标

这份手册对应 `Phase 0`，只做一件事：把当前项目的训练边界收紧，让第一版科学闭环先在数学主线上跑通。

本手册不直接改动 runtime 决策逻辑，不替代路线图，也不引入新的数据库规则。它的职责是把第一阶段的训练对象、训练意图、追问边界和验收标准写清楚，作为后续 prompt、activity、observation、mastery engine 和家长报告的共同约束。

配套代码入口：

- `content/goals/subgoal-playbooks.ts`
- `docs/ai-training-system-roadmap.md`

## 2. 第一阶段边界

### 2.1 主训练域

- 主训练域只保留 `math-thinking`

### 2.2 辅助训练域

- `logical-reasoning` 仅用于支撑数学中的判断、排除、预测
- `language-thinking` 仅用于支撑数学中的解释表达

### 2.3 暂不纳入科学闭环评估的域

- `creative-thinking`
- `strategy-thinking`
- `observation-induction`
- 其他未来扩展域

这些能力域仍可保留内容和入口，但在第一版闭环里不承担 mastery 判断主线，避免系统同时铺太宽。

## 3. 首批闭环试点子目标

第一版只围绕 3 个数学子目标建立训练闭环：

1. `pattern-recognition`
2. `quantity-comparison`
3. `strategy-planning`

选择理由：

- 三者都已存在于当前 goal tree 中，和现有活动模板对得上
- 能覆盖数学互动里最常见的三类核心能力：
  - 找规律
  - 比数量与追踪变化
  - 想后果与做策略
- 适合低龄儿童，且容易形成结构化 observation

## 4. 试点子目标训练卡

### 4.1 pattern-recognition

训练意图：

- 让孩子先观察变化，再描述规则，再预测下一步
- 不把“答对”当唯一目标，必须尽量采到解释证据

重点观察：

- 是否指出哪里在变
- 是否说得出规则
- 是否能预测下一项
- 是否能解释错误选项

典型错误：

- 靠猜，不解释
- 只看最后两个数
- 把局部相似误当完整规律
- 答对后也说不出“为什么”

提示梯度：

1. 提醒观察变化方向
2. 并列前后两步，让孩子比较变化是否一致
3. 给两个候选规则，让孩子判断更像哪一个

退出条件：

- 已采到“预测 + 解释”证据
- 稳定后可切换到数量关系的迁移题
- 明显疲劳时及时 wrap up

### 4.2 quantity-comparison

训练意图：

- 让孩子不仅能比较多和少，还能说出“差多少、怎么变、怎样一样多”

重点观察：

- 是否比较结果正确
- 是否表达差值或补足关系
- 是否跟得上数量变化
- 是否能解释数量关系

典型错误：

- 只会说谁多
- 混淆当前数量与变化量
- 两步变化后丢失状态
- 解释只复述题面

提示梯度：

1. 先分别数清楚
2. 改问“再给 1 个会不会一样多”
3. 给两个可能差值，让孩子说哪个更合理

退出条件：

- 已采到“比较结果 + 关系解释”证据
- 稳定后可切到策略规划，训练下一步预测
- 如果进入机械回答，停止无效刷题

### 4.3 strategy-planning

训练意图：

- 让孩子能看下一步或下两步结果
- 让孩子比较不同走法的后果
- 让孩子说明“为什么这样选”

重点观察：

- 是否预测局面后果
- 是否比较两种策略
- 是否解释选择理由
- 是否会在失败后调整

典型错误：

- 只看眼前一步
- 重复用失败策略
- 选对但不会解释
- 只说“感觉这样好”

提示梯度：

1. 先问“你选完以后会剩下多少”
2. 并列两个走法，比较后果
3. 给半结构结论，例如“如果你想把 3 的倍数留给对手，该怎么做”

退出条件：

- 已采到“预测后果 + 解释策略”证据
- 稳定后可进入 transfer check
- 高度依赖提示时不升难

## 5. 第一阶段的系统约束

### 5.1 AI 不能做什么

- 不能自己切换到不在试点范围内的主训练目标
- 不能只为了“更好玩”随意换训练点
- 不能越过提示梯度直接给答案
- 不能只追求聊天流畅，忽略证据采集

### 5.2 AI 当前可以做什么

- 在当前子目标范围内组织互动
- 用追问、对比、局部提示承接孩子回答
- 要求孩子解释原因
- 在达到退出条件后 wrap up 或切换到相邻迁移任务

### 5.3 第一阶段允许的教学动作

- `probe`
- `hint`
- `contrast`
- `ask_to_explain`
- `ask_to_predict`
- `transfer_check`
- `wrap_up`

## 6. 对后续 Phase 的接口要求

这份手册不是孤立文档，后续阶段要严格围绕这里补系统能力。

### 6.1 对 observation 的要求

至少要能记录：

- 当前 `goalId`
- 当前 `subGoalId`
- 当前难度
- 孩子原始输入
- 是否答对
- 用了几次提示
- 是否完成自解释
- 采到的证据类型

### 6.2 对 mastery engine 的要求

必须能回答：

- 当前孩子在某个子目标处于什么阶段
- 为什么继续当前难度、升难还是降难
- 下一步该复练、迁移，还是切换相邻子目标

### 6.3 对活动选择器的要求

活动选择顺序要从：

- 先挑好玩的活动

升级为：

- 先确定当前子目标
- 再根据当前子目标和掌握状态挑活动

## 7. 第一阶段验收标准

这一阶段完成后，不看“AI 是否更像真人”，而看下面几条是否成立：

1. 所有试点轮次都能明确映射到 3 个试点子目标之一
2. 每个试点子目标都能明确回答“这一轮到底在练什么”
3. AI 追问不再发散到无关训练点
4. 后续 observation 和 mastery 改造可以直接对齐这份训练卡
5. 家长报告未来能从这里倒推出训练逻辑

## 8. 当前结论

第一版科学闭环不追求“全能力域都覆盖”，而是要先把数学主线做窄、做深、做可解释。

只要这 3 个子目标的训练卡写硬，后续 observation、mastery engine、prompt 约束和 parent report 才有稳定锚点。没有这一步，后面的系统化改造会继续被 AI 的自由发挥稀释掉。
