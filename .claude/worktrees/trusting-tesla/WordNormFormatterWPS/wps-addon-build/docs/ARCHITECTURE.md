# WordNormFormatterWPS 架构图

## 1. 总体结构

```txt
┌─────────────────────────────────────────────┐
│ WPS Writer                                 │
│  ├─ Ribbon 按钮                            │
│  │   └─ 一键规范排版                       │
│  └─ Taskpane（加载本地 Web 页面）          │
│      └─ http://127.0.0.1:3100/taskpane/    │
└─────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│ taskpane 前端                              │
│  ├─ app.js                                 │
│  ├─ wpsAdapter.js                          │
│  ├─ promptBuilder.js                       │
│  ├─ workerClient.js                        │
│  └─ formatter.js                           │
└─────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│ Cloudflare Worker 代理                     │
│  ├─ 统一封装 OpenAI API                    │
│  ├─ 屏蔽大陆直连问题                       │
│  └─ 给最终用户一个可直接使用的代理入口      │
└─────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│ OpenAI 模型                                │
│  └─ 只做“段落结构识别 -> 样式分类”         │
└─────────────────────────────────────────────┘
```

## 2. 核心边界
- **AI 只输出结构标签，不输出改写后的文本**。
- 落地执行时只做：`段落索引 -> 套样式`。
- 若返回结构和原段落数量不一致，直接中止。
- 任何情况下不把 AI 文本回写进正文。

## 3. 关键文件职责

### `wps-addon/ribbon.xml`
- 定义插件在 WPS 顶部功能区的按钮
- 按钮文案：`一键规范排版`
- 点击后应触发任务窗格显示 / 聚焦

### `taskpane/index.html`
- 任务窗格主页面
- 大按钮 + 状态区 + 日志区

### `taskpane/src/wpsAdapter.js`
- 负责和 WPS JS API 对接
- 读取段落
- 应用 Heading 1 / 2 / 3 / 正文样式

### `taskpane/src/promptBuilder.js`
- 构造 AI Prompt
- 明确禁止改写文字，只允许返回样式标签

### `taskpane/src/workerClient.js`
- 调用 Cloudflare Worker
- 统一请求头 / 超时 / 响应校验

### `taskpane/src/formatter.js`
- 负责调度流程：
  1. 读段落
  2. 构造 prompt
  3. 调 Worker
  4. 校验结果
  5. 套样式

## 4. 推荐执行链
1. 打开 WPS 文字
2. 打开任务窗格
3. 点击“**一键规范排版**”
4. 插件读取段落
5. Worker 返回样式数组
6. 插件仅应用样式
7. 界面显示完成统计

## 5. 为什么这条路线更稳
- 不再走 Office Add-in 那套兼容链
- 直接按 **WPS 插件模型** 组织文件
- Web Taskpane + 本地服务的调试成本低
- Cloudflare Worker 保留不变，AI 后端可复用
