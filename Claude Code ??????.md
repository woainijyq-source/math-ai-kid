# AI 儿童脑力互动项目·Claude Code 开发任务拆解

## 1. 目标
本文件用于把产品需求转换成 Claude Code 可执行的开发任务序列。

原则：
- 一次只推进一个明确目标
- 每轮先做可运行闭环
- 优先原型验证，不先追求大而全

---

## 2. 第一阶段总体开发目标
先用 Claude Code 做出一个可运行的 Web 原型项目，并完成 3 个 AI-native 原型入口。

最终结果：
- 可以在 iPad / Mac 打开
- 有首页
- 有 3 个原型入口
- 每个原型至少有一个完整体验闭环
- 有基础奖励反馈

---

## 3. Claude Code 任务拆解

## 阶段 A：项目初始化
### A1. 初始化项目
任务：
- 创建 Next.js + TypeScript 项目
- 安装 Tailwind CSS
- 安装 Zustand
- 安装 Framer Motion
- 创建基础目录结构

验收标准：
- 本地能启动
- 首页能打开
- 基础目录清晰

### A2. 建立工程基础设施
任务：
- 配置 ESLint / Prettier（如需要）
- 建立 `types/`、`store/`、`lib/`、`components/`、`content/`、`prompts/`
- 建立基础样式变量

验收标准：
- 项目结构可扩展
- 没有明显工程混乱

---

## 阶段 B：页面骨架搭建
### B1. 首页骨架
任务：
- 搭建首页布局
- 放置“继续任务”“奖励”“家长入口”等区域
- 做 iPad 优先布局

验收标准：
- 首页可点击进入不同页面
- 视觉层级清晰

### B2. 任务页骨架
任务：
- 建立 `/play/opponent`
- 建立 `/play/co-create`
- 建立 `/play/story`
- 建立统一任务壳组件

验收标准：
- 3 个入口都能进入对应页面
- 页面有统一风格

### B3. 结果页与奖励页骨架
任务：
- 建立 `/play/result`
- 建立 `/rewards`
- 建立 `/parent`

验收标准：
- 基础页面跳转完整

---

## 阶段 C：AI 能力层封装
### C1. AI 服务接口封装
任务：
- 建立统一 AI 调用模块
- 预留 chat / stt / vision / summary 接口
- 所有前端调用统一走服务层
- 避免把 prompt、状态和业务逻辑写死在页面组件中，便于后续迁移到 iOS / iPadOS

验收标准：
- AI 接口调用结构统一
- 后续可换模型和 provider

### C2. Prompt 结构化
任务：
- 建立角色 prompt
- 建立追问 prompt
- 建立任务结果反馈 prompt
- 建立家长摘要 prompt

验收标准：
- Prompt 不散落在组件里
- 可以按角色和场景独立调整

---

## 阶段 D：实现 3 个核心原型
### D1. AI 对手型原型
任务：
- 做一个可轮流互动的策略小游戏
- AI 能回应、挑战、简单复盘
- 完成后进入结果页

验收标准：
- 至少有 1 个完整可玩回合
- AI 对手感成立

### D2. AI 共创型原型
任务：
- 做一个与 AI 共创规则/小游戏的页面
- 孩子给出一个规则
- AI 承接并继续推进
- 最后给出反馈结果

验收标准：
- 能感受到“我在和 AI 一起创造”
- 不是问答式页面

### D3. AI 剧情型原型
任务：
- 做一个带选择分支的剧情体验
- 至少有 2-3 个关键决策点
- AI 根据选择给出不同回应

验收标准：
- 分支差异感明确
- 剧情不是静态文本

---

## 阶段 E：语音与输入增强
### E1. 语音输入
任务：
- 实现麦克风录音入口
- 上传音频给后端识别
- 把识别文本接入现有任务流程

验收标准：
- 至少一个原型支持语音输入

### E2. 触摸与拖拽交互
任务：
- 为至少一个原型加拖拽或点选式互动

验收标准：
- iPad 上手感自然

---

## 阶段 F：奖励与状态管理
### F1. 用户状态模型
任务：
- 建立用户进度结构
- 建立已解锁状态结构
- 建立当前身份/等级结构

验收标准：
- 首页、结果页、奖励页能共享状态

### F2. 奖励反馈闭环
任务：
- 完成任务后触发奖励反馈
- 首页可见世界变化或身份变化

验收标准：
- 用户能感知“玩完后有变化”

---

## 阶段 G：家长视角
### G1. 家长页基础版
任务：
- 展示今日互动记录
- 展示最近偏好和亮点
- 展示基础设置入口

验收标准：
- 家长能看懂产品价值

---

## 5. 给 Claude Code 的工作方式建议

### 推荐方式
每次只下达一个小目标，例如：
- “先初始化项目骨架，不做业务逻辑”
- “先把首页和 3 个入口页面做出来”
- “先实现 AI 对手型原型最小闭环”

### 不推荐方式
不要一次让 Claude Code：
- 同时做完整产品
- 一次做完所有页面和逻辑
- 先生成大量内容再回头调整结构

---

## 6. 当前最推荐的第一条 Claude Code 指令

建议先让 Claude Code 做：

### 指令目标
初始化项目 + 建立首页和 3 个原型入口页面骨架

### 原因
这是最稳的起步方式：
- 能快速看到东西
- 能为后面原型开发铺路
- 不会一开始就陷入 AI 逻辑和细节

---

当前项目最合适的推进方式是：

**先把产品文档翻译成 Claude Code 能执行的 Sprint 任务，再逐轮开发。**
