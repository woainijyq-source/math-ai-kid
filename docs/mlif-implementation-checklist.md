# MLIF 项目内执行清单

## 用途

这份清单把 **微跃迁启思框架（MLIF）** 从理念拆成项目内可以逐项检查的执行标准。

适用场景：

- 改首页
- 改 session 互动
- 改 prompt
- 改 child signal
- 改 choices
- 改收尾方式
- 改家长页
- 改奖励页

每次改动完成后，都应该用这份清单自检一次，确认没有把产品又拉回“题库感”或“老师讲课感”。

## 总体检查

- [ ] 入口看起来像“开始一段小聊天”，而不是“选择一道题”。
- [ ] AI 的第一反应是接住孩子的话，而不是直接推进脚本。
- [ ] 每轮只增加一个新认知动作，不会连续抛两个以上新问题。
- [ ] 卡住时提供的是“两个思路方向”，不是标准答案。
- [ ] 收尾像“先聊到这里”，不是“任务结算”。
- [ ] 家长页优先显示孩子怎么想、下一句怎么接，而不是复杂训练统计。

## 1. 首页

目标：让孩子感到脑脑今天想来聊一个小问题，而不是要她去做题。

对应文件：

- `components/home/home-page.tsx`
- `app/page.tsx`

检查项：

- [ ] 主标题是“聊一个小问题”语气，不是“训练/挑战/做题”。
- [ ] 主按钮像“去聊一会儿”，不是“开始任务/进入训练”。
- [ ] 主题入口是轻量的聊天方向，不像能力域管理面板。
- [ ] 右侧预览卡更像“聊天预告片”，不是题目清单或固定追问流程。
- [ ] 首页不会向孩子暴露 readiness、评分、训练状态等内部信息。

红线：

- [ ] 不出现“请选择题目”“下一题”“训练线”“能力域成熟度”这类字眼。

## 2. Session 顶部与舞台区

目标：让孩子感觉自己正在和脑脑继续一段对话，而不是在完成流程。

对应文件：

- `components/agent/session-page.tsx`
- `components/agent/universal-renderer.tsx`
- `components/agent/input-bar.tsx`

检查项：

- [ ] 顶部主题卡像“今天想聊什么”，不是“当前任务信息”。
- [ ] 会话区默认聚焦当前轮，不让长历史淹没孩子注意力。
- [ ] 输入入口优先支持自然开口，底部输入栏只是兜底。
- [ ] 不向孩子暴露 scoring、mastery、evidence 等内部机制。

红线：

- [ ] 不出现“训练点”“推荐难度”“正式评分中”这类语气。

## 3. Prompt 系统

目标：让真实模型在运行时遵守 MLIF，而不是靠内容巧合表现正常。

对应文件：

- `prompts/agent-system-prompt.ts`
- `lib/daily/select-daily-question.ts`
- `prompts/modules/training-intent.ts`

检查项：

- [ ] prompt 明确写出：AI 不是题库，也不是老师。
- [ ] prompt 明确写出：先接住，再推进。
- [ ] prompt 明确写出：先具体、只推半步。
- [ ] prompt 明确写出：不要工作纸腔，不要讲课腔。
- [ ] prompt 明确写出：卡住时给两个方向，不给标准答案。
- [ ] prompt 明确写出：收尾要柔和，不是讲知识点总结。

红线：

- [ ] 不允许 prompt 鼓励 AI 使用“现在请回答”“正确答案是”“我们来学习”这种语气。

## 4. Child Signal

目标：让系统先判断孩子当前状态，再决定下一步怎么接。

对应文件：

- `lib/daily/child-signal.ts`

检查项：

- [ ] 至少能区分：简短、有理由、有创意、犹豫、跑题、不想继续。
- [ ] signal 的输出直接影响下一步动作，而不是只记录不用。
- [ ] signal 的解释描述是为“下一步怎么接”服务，而不是做学术标签。

红线：

- [ ] 不能出现无论孩子说什么都走同一套 follow-up 的情况。

## 5. 孩子语言镜像

目标：让 AI 听起来像真的在听，而不是机械复读。

对应文件：

- `lib/daily/child-language.ts`
- `lib/daily/mock.ts`

检查项：

- [ ] AI 优先复述孩子的一小段关键词，而不是整句长复读。
- [ ] 镜像后会马上接一个自然的问题，不会只重复内容。
- [ ] 柔性收尾会引用孩子的一点想法，但不会像“作业总结”。

红线：

- [ ] 不允许连续两轮都出现大段机械复读孩子原话。

## 6. Choices

目标：当孩子卡住时，给的是“两个思路”，不是“两个按钮”。

对应文件：

- `lib/daily/choice-scaffold.ts`
- `components/agent/choice-grid.tsx`
- `lib/daily/mock.ts`
- `lib/daily/select-daily-question.ts`

检查项：

- [ ] `show_choices` 的文案像“先看重复 / 先看变化”这种思路提示。
- [ ] choice 与当前场景相关，而不是全局通用按钮。
- [ ] choice 的 prompt 像真人会说的话，而不是系统说明。
- [ ] 当孩子犹豫时，choice 会帮助她更容易开口。

红线：

- [ ] 避免“继续”“再说一个”“给我提示”“下一步”这种空按钮成为主 choices。

## 7. 收尾

目标：结束时像“一段小聊天先收住”，而不是“任务完成结算”。

对应文件：

- `components/agent/session-page.tsx`
- `lib/daily/mock.ts`
- `store/agent-store.ts`

检查项：

- [ ] 收尾标题和文案都应柔和，不要像闯关完成。
- [ ] 收尾应强调“今天你想到了一点什么”，不是“完成度多少”。
- [ ] 如果保留完成度，它必须是次级信息，不是主视觉主文案。
- [ ] 收尾按钮像“再聊一个小问题”，而不是“再来一局”。

红线：

- [ ] 不要让结束弹层像游戏结算或训练统计页。

## 8. 家长页

目标：家长能在 10 秒内知道今天聊了什么、孩子怎么想、下一句怎么接。

对应文件：

- `app/parent/page.tsx`
- `app/api/parent/report/route.ts`
- `lib/data/session-log.ts`

检查项：

- [ ] 首屏优先显示：今天聊了什么 / 她是怎么想的 / 明天怎么接着问。
- [ ] 孩子原话要尽量能被保留下来。
- [ ] 深层训练数据是二级内容，不是首屏主内容。
- [ ] 家长页语气要帮助陪聊，而不是只呈现评估结果。

红线：

- [ ] 不要把家长页首屏做成训练后台仪表盘。

## 9. 奖励页

目标：奖励是“今天聊完后留下了一点变化”，不是“成就结算”。

对应文件：

- `app/rewards/page.tsx`

检查项：

- [ ] 页面语气像“今天留下了什么小变化”。
- [ ] 奖励重心是“愿意开口、愿意解释、愿意再想半步”。
- [ ] 页面能让孩子感觉今天聊完有一点点被看见，而不是只是拿到一个 badge。

红线：

- [ ] 不要做成只展示徽章列表的结算页。

## 10. Daily Seed / Theme Playbook

目标：场景种子只是开口的材料，真正的互动策略在主题 playbook 里。

对应文件：

- `content/daily/daily-question-bank.ts`
- `content/daily/theme-playbooks.ts`
- `lib/daily/select-daily-question.ts`

检查项：

- [ ] daily question 更像“场景种子”，不是固定脚本。
- [ ] theme playbook 负责定义主题动作、语气、边界。
- [ ] 系统根据 child signal 动态选择下一步动作。

红线：

- [ ] 不允许系统再次退回“固定三问硬执行”的模式。

## 11. 改动前后的最小自检

每次改完任意一个模块，至少问自己这 6 个问题：

- [ ] 这一处更像陪孩子聊，还是更像给孩子出题？
- [ ] 孩子的话有没有真的影响下一句？
- [ ] AI 有没有只推进半步？
- [ ] 卡住时给的是方向，还是答案？
- [ ] 结束时像“先聊到这里”，还是像“完成任务”？
- [ ] 家长看完之后，明天能不能自然接一句？

## 结语

如果某次改动让产品更像：

- 题库
- 老师讲课
- 工作纸
- 闯关结算

那它就偏离了 MLIF。

如果某次改动让产品更像：

- 一个会听孩子说话的脑脑
- 一个会轻轻追问的同伴
- 一个能帮孩子把想法说清楚一点点的对话场

那它就在往 MLIF 正确的方向前进。
