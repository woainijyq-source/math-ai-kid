# 内部试玩检查单

目标：在正式给孩子试玩前，用 10 到 15 分钟完成一轮内部检查，确认体验闭环和观察口径都成立。

## 基础准备

- 启动开发环境：`npm run dev`
- 跑一轮回归：`npm run smoke`
- 建议设备：iPad 横屏优先，宽度约 1024

## 必走路径

1. 首页进入 `AI 对手型`
2. 完成一轮对手试玩并进入结果页
3. 返回首页，确认世界状态和最近结果已变化
4. 进入 `AI 共创型`，用一句自然规则完成一次共创
5. 进入 `AI 剧情型`，连续做完一轮分支
6. 打开奖励页和家长页，确认结算与摘要都能读懂

## 体验检查

- 首屏是否能一眼看懂当前要做什么
- 主要按钮是否足够大，点击目标是否明确
- 聊天区滚动是否平稳，是否容易找到最新一条消息
- 结果页是否清楚回答“我做了什么”“世界变了什么”
- 家长页是否像观察记录，而不是考试报告

## 内容检查

- AI 对手型：AI 是否像在引导孩子观察规律，而不是直接讲答案
- AI 共创型：规则文案是否像孩子会说出来的话，而不是成人写给系统的话
- AI 剧情型：每次选择是否真有“我在改变故事”的感觉
- 奖励文案是否强调变化和成长，而不是积分和排名

## 观察记录

- 孩子会不会主动要求再来一轮
- 哪一类输入最自然：选项、语音、还是短文本
- 哪个反馈最能让孩子停下来继续看：角色回应、身份变化、还是世界变化
- 什么时候开始走神：输入太长、等待太久、还是反馈太弱

## 阻塞项

出现以下情况，先不要给外部试玩：

- 页面首屏看不懂当前任务
- 完成一轮后没有明显结果反馈
- 家长页摘要与刚刚的试玩不一致
- 聊天或语音接口偶发失败后无法恢复
- iPad 横屏下主交互区明显挤压或滚动异常
